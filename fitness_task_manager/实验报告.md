## 一、 实验内容、要求或目的

**1. 实验目的：**
* **语言基础与编码能力**：熟悉Go语言在小型控制台系统开发中的应用，掌握核心的语言特性，包括结构体定义（Struct）、切片操作（Slice）、包组织（Packages）等基础程序设计能力。
* **数据持久化机制**：深入理解数据模型与持久化原理，掌握利用Go标准库 `os` 和 `encoding/json` 实现对象数据的序列化内存写入与硬盘文件的映射保存。

**2. 实验任务与内容：**
本次实验采用Go语言设计并实现了一套基于命令行的**“个人健身训练任务管理系统”**。系统重点围绕“任务”实体展开数据的生命周期管理，实现了如下功能：
* **基础功能**：独立设计包含任务ID、名称、描述、状态、创建时间及优先级的完整数据结构。实现健身任务的创建、结构化查询、状态修改以及任务删除。

* **加分项实践（防篡改操作追踪）**：独立实现了一套基于“链式日志”的记录系统。每一次影响持久化数据的操作（增/删/改）均会生成一条摘要，附加时间戳并结合上一条记录的哈希针（PrevHash）通过 `SHA-256` 算法加密生成本块数字指纹。最后提供日志独立校验引擎，用于鉴定底层 `json` 文本数据是否曾受到外部非法篡改。

**1. 如何描述需求与基础框架的搭建：**

* **AI 的帮助**：AI并没有直接扔给我一堆杂乱的代码，而是像导师一样先向我解释了Go语言中没有 `Class`（类），而是通过 `struct`（结构体）来聚合数据。AI给出了 `FitnessTask` 和 `LogBlock` 的结构原型，并重点提醒我：**要让 `encoding/json` 包成功导出字段，结构体名的首字母以及字段的首字母都必须大写**；同时可以利用反引号（如 `` `json:"id"` ``）来映射JSON文件里的键名。这个特性解答了我之前自学时的很大困惑。

**2. 核心算法探究与疑难Bug调试：**
* **遇到的问题（删除数组元素的坑）**：在实现删除功能时，我最初尝试将数组中对应该任务的索引置空，但总是导致访问越界或后续打印空指针错误。
* **AI 的指导**：AI教会了我Go独有的切片拼接语法 `tasks = append(tasks[:i], tasks[i+1:]...)`。AI非常生动地帮我剖析了内存模型：切片在Go中是指向底层连续数组的窗口，上述语法的作用就是截走前半段和后半段，然后利用 `...` 操作符将后半段元素打散重组到前半段，实现了优雅的高效删除。
* **加密链算法调整**：在处理“链式日志”加分项时，AI帮我理清了 `crypto/sha256` 包的调用流程。最初用 `time.Now().String()` 拼接内容造成每次重新校验都哈希不匹配，AI指导我改用 `time.Format(time.RFC3339)` 输出了固定格式，完美攻克了哈希碰撞断裂的Bug。

**3. 对AI生成结果的修改或调整（个人重构）：**

### 1. 利用 Struct Tag 实现数据模型定义与JSON化

type FitnessTask struct {

	Deadline    time.Time `json:"deadline"`    // 扩展项：截止时间

```

### 2. 基于切片的CRUD核心逻辑（以删除操作为例）

### 3. 链式防篡改密码学记录引擎（加分项核心）
系统利用前置哈希指针，将每次发生的数据变动串联。一旦底层JSON被直接利用记事本强行修改，程序下一次开启便能自动拦截。
```go
// Hash 发生器：接收时间、动作并包含前置块Hash组合文本基料加密
func calculateHash(index int, timestamp time.Time, summary, prevHash string) string {
	record := strconv.Itoa(index) + timestamp.Format(time.RFC3339) + summary + prevHash
	
	h := sha256.New()
	h.Write([]byte(record)) // 必须将 string 换算至 []byte
	return hex.EncodeToString(h.Sum(nil))
}

// 防篡改校验核心循环器
func verifyChain() bool {
	for i := 1; i < len(blockchain); i++ {
		prev := blockchain[i-1]
		curr := blockchain[i]
		
		// 1. 本块存放的 PrevHash 是否与上一实体Hash一致 (防抽轨)
		if curr.PrevHash != prev.Hash { return false }
		// 2. 本块现存正文重新结算是否匹配现存Hash (防防篡改内容)
		if curr.Hash != calculateHash(curr.Index, curr.Timestamp, curr.Summary, curr.PrevHash) { 
            return false 
        }
	}
	return true
}
```

### 第一次习题练习（第四章：控制结构）

**1. 第16题：水仙花数**
* **题目说明**：编程输出所有的“水仙花数”（三位的十进制整数，其各个数位上数字的立方和恰好等于这个整数本身）。
* **解题思路**：使用 `for` 循环遍历100至999，在循环内分别使用整除（`/`）和取余（`%`）操作提出百位、十位和个位，运算立方和并与原数字进行对比验证输出。
* **核心代码**：
```go
package main

import "fmt"

func main() {
	fmt.Print("所有的水仙花数有: ")
	for i := 100; i < 1000; i++ {
		a := i / 100       // 分离百位
		b := (i / 10) % 10 // 分离十位
		c := i % 10        // 分离个位

		if a*a*a + b*b*b + c*c*c == i {
			fmt.Printf("%d ", i)
		}
	}
	fmt.Println()
}
```

**2. 第18题：最大公约数与最小公倍数**
* **题目说明**：输入两个正整数 m 和 n，求其最大公约数 gcd 和最小公倍数 lcm。
* **解题思路**：最快方案为应用辗转相除法配合Go多重赋值特性（`m, n = n, m%n`）回避老式的临时临时变量(temp变量)操作。最小公倍数套用两数乘积除以最大公约数的公式求出。
* **核心代码**：
```go
package main

import "fmt"

func gcd(m, n int) int {
	for n != 0 {
		m, n = n, m%n // Go语言多重赋值迭代
	}
	return m
}

func lcm(m, n int) int {
	return (m * n) / gcd(m, n)
}

func main() {
	var m, n int
	fmt.Print("请输入两个正整数 m 和 n: ")
	fmt.Scanf("%d %d", &m, &n)

	fmt.Printf("最大公约数是: %d\n", gcd(m, n))
	fmt.Printf("最小公倍数是: %d\n", lcm(m, n))
}
```

### 第二次习题练习（第五章：函数与方法）

**3. 第21题：使用闭包创建偶数生成器**
* **题目说明**：使用闭包创建一个偶数生成器 `evenGenerator()`。
* **解题思路**：在一个函数内定义闭包函数作为返回值。在外部方法内预置状态变量 `i := 0`，匿名内部闭包执行对该变量的读写修改运算并使得其生命期被持久锁定。
* **核心代码**：
```go
package main

import "fmt"

func evenGenerator() func() int {
	i := 0 // 外部环境变量，由下方闭包捕获生命期
	return func() int {
		ret := i
		i += 2 // 每次调用自增保存2
		return ret
	}
}

func main() {
	nextEven := evenGenerator() // 取得闭包生成器句柄

	fmt.Println(nextEven()) // 0
	fmt.Println(nextEven()) // 2
	fmt.Println(nextEven()) // 4
}
```

### 第三次习题练习（第六章：数据容器）

**4. 第13题：切片追加与解包**
* **题目说明**：编程实现：为切片 s 追加一个新切片 `[]int{1, 2}`。
* **解题思路**：切片追加利用内置函数 `append()`，但传入另一个切片时需要注意直接添加会导致类型冲突。使用Go内置的展开语法糖 `...` 将目标切片“解包”为单个可迭代参数再合并。
* **核心代码**：
```go
package main

import "fmt"

func main() {
	s := []int{8, 9, 10}
	newSlice := []int{1, 2}

	// 核心步骤：使用 ... 将新数组解包成独立参数传入 append 中
	s = append(s, newSlice...)
	fmt.Printf("追加后的切片: %v\n", s)
}
```

**5. 第20题：列表（List）的操作与遍历**
* **题目说明**：在列表 lt 的头部增加元素 5，在其尾部依次增加 3 个元素 "b"、"c"、"a"。遍历该列表，输出其元素的值。
* **解题思路**：引入双向链表核心模块 `container/list` 构造列表，随后通过暴露的方法 `PushFront` (首部) 以及 `PushBack` (尾部) 追加，指针遍历采用前驱遍历器并结合泛型方法获取。
* **核心代码**：
```go
package main

import (
	"container/list"
	"fmt"
)

func main() {
	lt := list.New() // 实例化

	lt.PushFront(5)     // 头部追加
	lt.PushBack("b")    // 尾部推入
	lt.PushBack("c")
	lt.PushBack("a")

	fmt.Print("链表遍历输出: ")
	// 通过 Front 方法提取首结点做为其实游标，并沿 Next() 不断检索
	for e := lt.Front(); e != nil; e = e.Next() {
		fmt.Printf("%v ", e.Value)
	}
	fmt.Println()
}

**（2）链式防篡改记录系统（算法核心）：**
利用前置哈希指针，将文件修改动作串联。任何通过记事本的非法修改，都会在下一次运行时被拦截。
```go
// 核心哈希算法片段：接收当前时间、摘要内容和前驱节点指纹
func calculateHash(index int, timestamp time.Time, summary, prevHash string) string {
	// 拼接长字符串组合全部信息基料
	record := strconv.Itoa(index) + timestamp.Format(time.RFC3339) + summary + prevHash
	
	h := sha256.New()           // 初始化散列加密对象
	h.Write([]byte(record))     // Go中要求将字符串强转为字节切片 []byte
	return hex.EncodeToString(h.Sum(nil)) // 转产为可视化的16进制指纹
}

// 篡改校验器片段
func verifyChain() bool {
	for i := 1; i < len(blockchain); i++ {
		prev := blockchain[i-1]
		curr := blockchain[i]
		// 规则1：当前链结的“挂钩”是否与上一节实体吻合
		if curr.PrevHash != prev.Hash { return false }
		// 规则2：实体内容是否被恶意更改（用现存数据重新跑一遍散列并对质）
		if curr.Hash != calculateHash(curr.Index, curr.Timestamp, curr.Summary, curr.PrevHash) { return false }
	}
	return true
}
```
